Architecture Overview

Uber's distributed microservices architecture handles real-time matching, geospatial computation, payment processing, and global event streaming at scale.

📱 Client Layer
iOS / Android / Web (React Native + Swift/Kotlin) — Native SDKs for GPS accuracy and offline resilience. SDK calls gRPC services for low-latency ride matching.
🌍 CDN / Edge
Cloudflare + AWS CloudFront — Map tiles, static assets, and app bundles cached at 50+ PoPs globally. Edge compute (Cloudflare Workers) handles auth token validation and rate limiting without hitting origin.
⚡ Load Balancer / API Gateway
AWS ALB + Kong API Gateway — Layer-7 load balancing across API fleet. Kong handles auth, rate limiting, request routing, and observability middleware. gRPC traffic proxied via Envoy.
🔧 Application / Microservices Layer
Go + Python + Java (Kubernetes on EKS) — Key services:
Dispatch Service — Real-time driver-rider matching using Hungarian algorithm
Geo Service — H3 hexagonal grid for geospatial queries (driver availability)
Surge Pricing — Real-time ML demand/supply balancing
Trip Service — Trip lifecycle state machine (requested → active → completed)
Payments Service — PCI-DSS compliant, integrates Stripe + Braintree
Notification Service — Push (FCM/APNs), SMS (Twilio), email via SES
ETA Service — ML-powered arrival time predictions using historical + live traffic
📨 Event Streaming / Async
Apache Kafka (MSK on AWS) — GPS telemetry, trip events, payment events all flow through Kafka. Flink processes GPS streams for real-time driver location aggregation. Enables decoupled microservices communication.
🗄 Database Tier
PostgreSQL (RDS) — Trips, users, payments (ACID compliance required)
Apache Cassandra — GPS traces, time-series data (write-optimized, geo-partitioned)
Redis Cluster — Driver live location (geo-spatial), session cache, surge multipliers
Elasticsearch — Address autocomplete, POI search, driver search
S3 + Glacier — Trip receipts, audit logs, ML training data archival
🤖 ML / AI Layer
AWS SageMaker + Vertex AI — ETA models, surge pricing models, driver fraud detection. Feature store (Feast) provides low-latency feature serving. Models retrained daily on Spark clusters.
📊 Observability
Datadog + Prometheus + Grafana + PagerDuty — SLI/SLO tracking for matching latency (<500ms p99), GPS ingest lag (<1s), payment success rate (>99.9%). Distributed tracing via Jaeger.
